# 🛠 V7.5.1 핫픽스 패치 플랜 (For Sonnet)

이 문서는 2026-03-23 새벽에 발견된 라이브 QA 이슈들을 해결하기 위한 구체적인 액션 플랜입니다. Claude 3.7 Sonnet 에이전트가 이 문서를 컨텍스트로 삼아 바로 코드를 수정할 수 있도록 작성되었습니다.

## 📸 패치 1: 우주맵 이미지 렌더링 정상화 (이슈 #2, #8)
- **목표**: 줌 아웃 상태나 바텀시트가 열린 상태에서도 아티스트 사진이 사라지지 않게 한다.
- **대상 파일**: `src/components/GraphCosmos.tsx` (약 400~520행 부근)
- **작업 내용**:
  1. `lod === "mid"` 조건에서도 `node.image`가 존재하면 캐시된 이미지를 로드하여 원 안쪽에 그려주도록 캔버스 렌더링 로직 추가. (이니셜 렌더링하는 분기점 수정)
  2. `lod === "close"` 조건에서 이미지를 그리는 `&& (isFocused || isHop1 || isMajor)` 제약 조건을 해제. 이미지가 있으면 `node.image` 여부만으로 언제든 그리도록 변경.
  3. 바텀시트가 올라오면서 viewport 높이가 줄어들어 LOD가 강제로 MID 이하로 떨어지는 현상을 보정하거나, MID에서도 이미지가 나오기 때문에 자연스럽게 해결되는지 확인.

## 🎵 패치 2: iTunes 동명 곡 오작동 수정 (이슈 #3)
- **목표**: 밴드 "라이너스의 담요" 클릭 시, 다른 가수의 "라이너스의 담요"라는 노래를 가져오지 않도록 한다.
- **대상 파일**: 미리듣기를 가져오는 백엔드 API 라우트 또는 클라이언트 로직 (예: `src/app/api/itunes/search/route.ts` 등 `fetch` 로직)
- **작업 내용**:
  1. iTunes Search API (`https://itunes.apple.com/search`) 호출 파라미터 확인.
  2. 현재 단순 곡명 검색(`term=${artistName}`)으로 되어 있다면, 일치하는 아티스트명(`artistName` 필드)인지 교차 검증하는 필터링 로직 추가.
  3. 검색 시 `attribute=artistTerm` 또는 `entity=song` 일 때 결과 객체의 `artistName`을 정규화(공백, 원음 등)해서 대상 아티스트와 일치하는 트랙만 반환하도록 수정.

## 🔗 패치 3: 엣지(관계선) 팝업 복구 (이슈 #4)
- **목표**: 우주맵 상의 선(엣지)을 클릭했을 때 관계 타입/근거를 보여주는 팝업 작동 복구.
- **대상 파일**: `src/components/GraphCosmos.tsx` 및 관련 UI 컴포넌트
- **작업 내용**:

  2. `SheetState === "expanded"` 상태일 때 클릭 이벤트를 무시하거나, 투명 오버레이 패널에 막혀 엣지 클릭이 전달되지 않는지 DOM 레이어 / `pointerEvents` 문제 해결.

## 🔍 패치 5: 전체보기(ZoomToFit) UX 개선 (이슈 #6)
- **목표**: `⊞` 버튼을 눌렀을 때 먼지처럼 줌아웃되는 현상 개선.
- **대상 파일**: `src/components/GraphCosmos.tsx` (`handleZoomToFit` 함수)
- **작업 내용**:
  1. 무조건 전체 그래프를 fit 하는 것이 아니라, 현재 포커스 아티스트가 있는 경우 **로컬 줌(예: 1~2촌 범위)**으로 fit 하도록 로직 분기 방안 마련.
  2. 또는 ForceGraph의 `zoomToFit` 대신 한계 줌 레벨(`.zoom(0.5, 800)`) 등으로 고정.
