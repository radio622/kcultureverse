## 📅 Phase 1: 치명적 시각 및 데이터 버그 조기 진압 ✅ 완료

### 1-1. 🌌 맵 상 엣지(관계선) 실종 버그 복구 ✅
- `weight >= 0.5` 강관계만 줌아웃 시 유지 → 헤어볼 방지 + 가독성 회복
- `linkColor`, `linkWidth` 콜백에서 줌 스케일 반응형 가시성 적용

### 1-2. 🤖 듀얼 하베스터 봇 (Spotify 401 에러) 패치 ✅
- `SpotifyTokenManager` 클래스: 만료 5분 전 자동 갱신 + 401 즉시 강제 갱신
- `progress.json` 원자적 저장으로 체크포인트 안전 보장
- **재가동 대기 중**: 44번째 아티스트(A.TRAIN)부터 이어하기 준비 완료

### 1-3. 👆 엣지(관계선) 클릭 팝업 히트박스 교정 ✅
- PC: 1.05배 → 모바일: 1.15배 히트영역 분리 적용
- 줌아웃 임계값 0.4 → 0.25로 완화 (줌아웃에서도 팝업 표시)
- 노드 근접 30px 이내 클릭은 노드 우선으로 양보

## 📅 Phase 3: V7.7의 혁신, 신규 바이럴 및 지형지물 확장

### 3-1. 🚀 "A로부터 B에게" 우주 여정 자동 재생 및 OG 공유 ✅ 완료

구현 완료 내역:
- `useJourneyPlayer` 훅: Dijkstra 경로 순회 + 미리듣기 자동 재생
- Bird's Eye View (1.8초 시네마틱 줌아웃) → 4초 전류 감상 → 줌인 여행 시작
- 전류 애니메이션: 일정 속도(0.06px/ms), from→to 방향 보장 (좌표 스왑 방식)
- `/universe?artist=A&to=B` URL 및 SNS 공유 (조사 자동 처리)
- 여정 진행 바 UI + 도착 토스트
- RAF 루프 `refresh()` + 30fps 스로틀 성능 최적화
- 모바일 공유 버튼 overflow 방지 (`maxWidth`, 여정 모드 시 정보수정 숨김)

⚠️ **잔여 미완 항목**:
- `A→B` 여정용 OG 썸네일 자동 생성 (현재는 단일 아티스트 OG만 지원)
- 여정 종료 후 `journeyShareUrl` 자동 초기화
- 여정 중 바텀시트 자동 접힘 처리

### 3-2. 🌐 다차원 관계망 (가족, 학연, 소속사) 엣지 확장 및 유저 검증 ❌ 미착수
- **엣지 타입 확장**: V7.5에서 18종 완료 (FAMILY, ALUMNI, AGENCY_MATE 등)
- **액션 (남은 것)**:
  - 유저가 "관계 제안하기" 폼을 통해 증거(기사 URL 등)와 함께 제안하는 시스템 개발.
  - 관리자(`/admin`) 대시보드 검토/승인 — `edge_suggestions` Supabase 테이블 설계
  - V5Edge 타입 확장 업데이트 및 제안 열람 UI

### 3-3. 🎨 아티스트별 앨범/테마 연동형 우주 개인화 렌더링 ❌ 미착수
- **개념**: 특정 아티스트 포커스 시, 우주의 바탕색과 렌더링 입자들이 해당 앨범/가수의 테마색과 분위기에 맞춰 변화.
- **액션**:
  - 이미지 퀀타이제이션 알고리즘으로 앨범 커버의 Main Color 추출.
  - 추출된 색상을 우주 Canvas 글로벌 배경색(그라데이션 및 파티클 색상) 객체에 동적 바인딩.

### 3-4. 💜 정회원 엣지 '하트 투표' 물리적 지형 변화 ❌ 미착수
- **개념**: 정회원이 하루 100개씩 좋아하는 두 노드 사이의 거리에 투표("가깝게!" / "멀게!")하여 실제 엣지 가중치(`weight`)가 수정되는 팬덤의 장.
- **액션**:
  - Supabase `heart_votes` 테이블 설계 및 투표 API 누적 로직 구현.
  - 매일 자정 또는 누적 티핑포인트 도달 시 `layout.json` 상의 거리를 변화시키는 CRON 배치 구축.

### 3-5. 🎤 아티스트 공인 인증(✅) 배지 및 커스텀 마케팅 링크 ❌ 미착수
- **개념**: 가수 본인 요청 시 오피셜 배지 획득 및 노드 내 콘서트 단독 예매 링크 삽입 기능.
- **액션**: 노드 뱃지 렌더러 추가 및 JSON 페이로드에서 `verified_links` 배열 확장. (기획 및 운영 정책 단 우선 수립)
