# 🌌 K-Culture Universe V5.4: Organic & Egalitarian Network Reconstruction

> **작성일**: 2026-03-21
> **현황 진단**: 이전 V5.3 버전의 치명적인 "허브/위성 계급화 구조" 및 "2촌 다이렉트 연결 크롤링 돌진 에러"를 전면 폐기하고, 본래 기획 목적이었던 **모든 아티스트가 평등하고 유기적으로 무한정 이어지는 거미줄 형태의 우주**를 재구축하기 위한 마스터플랜.

## 🛑 1. 기존 V5.3의 치명적 문제점 진단 (Root Cause Analysis)

### 1) 설계의 패착: 아티스트 계급화 (Tier 0, 1, 2)
- **문제점**: 62명의 특정 아티스트를 `Hub(Tier 0)`로 강제 지정하고, 나머지 아티스트를 `Satellite(위성, Tier 1/2)`로 종속시켜버렸음.
- **결과**: 유기적으로 엮여야 할 K팝 신(Scene)이 62개의 고립된 섬(파벌)으로 쪼개져버렸으며, 유저에게 심각한 불쾌감과 이질감을 유발함.

### 2) 코딩 에러: 잘못된 간선(Edge) 연결 로직
- **문제점**: `빅뱅`의 앨범에서 `테디(1촌)`가 프로듀싱하고, 테디가 `블랙핑크(1촌의 1촌=2촌)`와 작업한 경우, 정상적인 간선은 `[빅뱅] - [테디] - [블랙핑크]` 로 징검다리처럼 이어져야 함.
- **코딩 오류**: 크롤러가 편의를 위해 `[빅뱅]-[테디]`, 그리고 `[빅뱅]-[블랙핑크]` 식으로 발견된 모든 관계를 최초 시작점(빅뱅)에 다이렉트로 갖다 꽂아버림.
- **결과**: 기괴한 방사형(성게 반쪽 모양) 군집들이 형성되고, 정작 빅뱅과 블랙핑크를 잇는 중간 다리(테디)의 역할은 시각적으로 소실됨.

### 3) 렌더링 뷰 오류: 첫 화면부터 빽빽하고 진한 선
- **문제점**: 유기적 거미줄은 거시적으로(Zoom-out) 보았을 때 선이 보이지 않고 별빛(점)만 보여야 아름답고 깔끔함. 현재는 모든 관계선이 원색으로 진하게 렌더링됨.

## 🗺 3. 단계별 개발/수정 플랜 (Action Plan)

### Phase 1: 데이터 모델 '평탄화(Flatten)' 및 계급 폐지
1. 데이터베이스(혹은 정적 JSON) 구조 전면 수정: `tier` 구분 철폐.
2. 모든 아티스트는 오직 고유의 식별자(Spotify ID / MBID)를 갖는 동등한 `Node` 객체가 됨.
3. `/data/hub/*.json` (위성 종속형 데이터) 관리 방식 폐기. 모든 데이터를 단일 (혹은 분할된) 범용 `Nodes` / `Edges` 풀로 통합.

### Phase 2: '징검다리' 엣지 크롤러 전면 재구성 (가장 중요)
기존의 `crawl-featured-credits.ts`와 `crawl-writer-credits.ts`를 완전 폐기하고 `bfs-crawler.ts`로 통합 재작성:
1. **BFS Queue 기반 탐색**: 초기 Seed 아티스트 N명을 큐에 넣고 돌기 시작.
2. **관계 정확도 보장**: A를 탐색하여 B를 발견하면, 간선은 반드시 `source: A, target: B`로 기록.
3. 큐에서 팝한 B를 탐색하여 C를 발견하면, 간선은 반드시 `source: B, target: C`로 기록 (A에 연결하는 에러 원천 차단).
4. **블랙핑크-빅뱅 단절 해소**: 크롤링이 진행되며 B(테디 등)를 매개로 자연스럽게 거대한 거미줄이 형성됨.

### Phase 3: D3 물리 엔진 레이아웃 튜닝 (1,000+ 아티스트 대응)
- 아티스트가 평등해지면, d3-force의 반발력(ManyBody)과 스프링 인력(Link) 만으로 거대한 은하의 모양이 잡힘.
- 중심 허브가 없으므로 1,000명이 넘어갈 때 그래프가 산산조각 나지 않도록 중심 인력(`forceCenter` 및 `forceRadial`)을 적절히 조율.
- 베이킹 스크립트(`build-universe-v5.ts`) 로직 수정.

### Phase 4: 시각적 폴리시 (첫인상 개선)
1. 첫 화면 로드 시 모든 간선(Edge)의 Opacity를 5% 이하 유령선 레벨로 낮춤 (별만 보이도록).
2. 유저가 화면의 특정 영역을 확대(Zoom In)하거나 아티스트 마우스 오버/클릭 했을 때만, 1/2촌 선 굵기와 불투명도가 살아나는(Glow) 효과 적용.

## ⚡ 4. 알고리즘적 병목 방지 대책 (1,000~5,000명 대응)

1. **d3-force 브라우저 로딩 회피 (Offline Baking)**
   - 브라우저 클라이언트 연산 시 `O(N^2)` 프레임 드랍 발생.
   - V5.3의 오프라인 Node.js 환경 좌표 사전 연산 후 전송 방식 유지.
2. **BFS 크롤링 중복 및 무한루프 방지**
   - 탐색 Node `Set(MBID)`과 양방향 엣지 해시키 `Hash(min, max)`를 필수 운용하여 중복 엣지 생성 완벽 차단.
3. **Canvas Culling**
   - 브라우저 화면(Viewport) 밖으로 나간 수천 개의 점/선 객체는 연산(`fillRect`/`arc`)을 즉시 무시하여 60프레임 방어.
