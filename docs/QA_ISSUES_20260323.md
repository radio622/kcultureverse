## 🔴 미해결 — 우선순위 높음

### 2. 다수 아티스트 사진 미노출 — ✅ 원인 확인됨!
- **증상**: BLACKPINK에서 JENNIE, JISOO, ROSÉ 등 주요 아티스트의 카드에 사진 대신 이니셜만 표시
- **데이터 현황**: `v5-details.json` → 1,213명 중 756명(62%)은 `image` 필드 존재
- **🔍 근본 원인 (코드 확인 완료)**: `GraphCosmos.tsx` 503행의 렌더링 조건이 너무 엄격함
  ```javascript
  // CLOSE LOD (줌 ≥1.5배)에서도 3가지 경우에만 이미지 렌더링
  if (node.image && (isFocused || isHop1 || isMajor)) { ... drawImage ... }
  ```
  - **FAR** (줌 <0.4): 점만 렌더링 → 이미지 ❌
  - **MID** (줌 0.4~1.5): 원+이니셜 렌더링 → 이미지 ❌ (여기가 대부분의 사용 줌 레벨!)
  - **CLOSE** (줌 ≥1.5): focused/hop1/major만 → 대다수 노드 이미지 ❌
- **결론**: 데이터에 이미지가 있어도, **MID 줌에서는 절대 이미지를 안 그리고, CLOSE에서도 포커스/1촌/메이저만** 그리므로 거의 안 보이는 것
- **해결 방향**: 
  - MID LOD에서도 이미지가 있는 노드는 사진을 표시하도록 렌더링 조건 완화
  - CLOSE LOD에서 `isFocused || isHop1 || isMajor` 조건 제거 → 이미지가 있으면 무조건 표시
  - 성능 우려 시 화면에 보이는 노드 수 기준으로 동적 LOD 적용

### 3. "라이너스의 담요" 클릭 시 잘못된 미리듣기 재생
- **증상**: 밴드 "라이너스의 담요"를 클릭했는데, 다른 아티스트의 "라이너스의 담요"라는 **노래**의 프리뷰가 재생됨
- **원인**: iTunes Search API가 아티스트명과 동명의 노래(trackName)를 우선 매칭하는 문제
- **해결 방향**: iTunes 검색 시 `entity=musicArtist`로 아티스트를 먼저 찾고, 해당 아티스트의 곡을 별도 검색하는 2단계 방식

### 4. 엣지(관계선) 클릭 시 관계 설명 팝업 미동작
- **증상**: 엣지 라인을 클릭해도 관계 타입/근거를 보여주는 팝업이 뜨지 않음
- **원인**: 미조사 — `edgePopup` 로직 또는 클릭 이벤트 핸들러 확인 필요
- **확인 사항**: `GraphCosmos.tsx`의 `onLinkClick` 핸들러 및 `edgePopup` state 확인
- **참고**: 이전에 잘 됐었다고 유저 증언 → 리그레션 가능성

## 🟡 미해결 — 중간 우선순위

### 5. 우측 상단 로그인(유저 아바타) 아이콘 안 보임
- **증상**: 우주 화면에서 우측 상단에 있어야 할 로그인/프로필 아이콘이 보이지 않음
- **코드 확인**: `UserAvatar` 컴포넌트는 `position: fixed, top: 16, right: 66, zIndex: 200`으로 렌더링 중
- **원인 추정**: 
  - Auth.js 세션 로딩 중 렌더링이 안 되는 것일 수 있음
  - 또는 다른 레이어에 가려진 것
- **확인 방법**: 브라우저 개발자 도구에서 `#user-avatar-btn` 존재 여부 확인

### 6. ⊞ (전체 우주 보기) 버튼 UX 개선 필요
- **증상**: 클릭하면 1,392명이 한 화면에 담기면서 먼지처럼 보임 → 사용자에게 무의미
- **현재 동작**: `zoomToFit(800ms, padding 60px)` — 정상 동작이지만 UX가 좋지 않음
- **개선 방향**: 
  - 최소 줌 레벨 제한 추가 (너무 작게 줄어들지 않도록)
  - 또는 현재 포커스 아티스트의 1~2촌까지만 fit하는 "로컬 줌" 방식으로 변경

### 7. 미리듣기 30초 후 자동 넘어가기 미동작 (확인 필요)
- **증상**: 미리듣기가 30초 후 끊기고 다음 곡으로 안 넘어감
- **설계 의도**: 자율주행(Auto-Warp) 모드에서만 릴레이 재생됨. 수동 클릭 시에는 1곡만 재생 후 정지가 정상.
- **확인 필요**: 유저가 자율주행 모드에서 테스트했는지 여부
- **해결 방향**: 자율주행 아닌 경우에도 같은 아티스트의 다른 곡을 이어 재생하는 옵션 검토

### 8. 카드덱 열리면 우주맵 아티스트 사진이 전부 사라짐
- **증상**: 바텀시트 닫힘 상태에서는 아이유, 서태지, ROSÉ 등의 사진이 정상 표시됨. 아티스트 클릭으로 카드덱(바텀시트)이 올라오면 우주맵의 모든 아티스트 사진이 사라지고 이니셜/텍스트만 남음
- **원인 추정**: 바텀시트가 화면 하단을 차지하면서 Canvas 뷰포트 높이가 줄어들고, `react-force-graph`의 줌 레벨이 변동하여 LOD(Level of Detail)가 Close(이미지 표시) → Mid(텍스트만 표시)로 전환됨
- **해결 방향**: 
  - LOD 판정 시 실제 줌 레벨을 참조하되, 바텀시트에 의한 뷰포트 축소를 보상
  - 또는 바텀시트가 열려도 Canvas 높이를 줄이지 않고 오버레이로 처리
- **스크린샷**: 왼쪽(카드덱 닫힘: 사진 보임) vs 오른쪽(카드덱 열림: 사진 전부 사라짐)
