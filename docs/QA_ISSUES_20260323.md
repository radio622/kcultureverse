## 🔴 최우선 해결 과제 (Blockers)

### #1 백그라운드 스크립트 401 Unauthorized 무한 반복
- **증상**: `v7.5-dual-harvester-draft.py` 실행 시 Spotify API 검색 단계에서 `HTTP Error 401: Unauthorized` 발생 후 10분 대기 늪에 빠짐.
- **원인**: Python 스크립트 내 Spotify 토큰 갱신(Client Credentials Flow) 로직 부재 혹은 `.env.local` 키 만료.
- **다음 작업 조치**: 
  - Python 스크립트 내 `get_spotify_token()` 등의 갱신 로직을 추가하여 무중단 크롤링 재개 (PID는 이미 kill 처리함).
  - 현재 44명까지 성공한 `scripts/progress.json` 내역을 바탕으로 45번째 아티스트부터 이어받기 적용.

## 🟡 차순위 개선 과제 (UI/UX & Features)

### #2 수동 미리듣기 곡 연속 재생 (자율주행 유사 기능)
- **증상**: 바텀시트에서 카드(미리듣기)를 수동으로 클릭하면 한 곡(30초)만 재생되고 정지됨.
- **요구사항**: 사용자가 수동 모드 상태여도, 노래가 끝나면 해당 카드의 다음 곡이나 연관된 다음 아티스트 카드로 넘어가며 계속 재생되어야 함 (마치 라디오처럼 연속재생).
- **다음 작업 조치**: `MiniPlayer.tsx`와 백그라운드 오디오 `useAudio/useAutoWarp` 로직을 융합하여 수동 큐(Queue) 시스템 구축.

### #3 바텀시트 및 기타 잔존 UI 엉망 현상
- **증상**: padding/margin 수치를 줄여 여백을 54px 가량 제거했음에도 불구하고, 화면 해상도나 기기에 따라 카드덱 안의 버튼이 짤리거나 `정보 수정 제안` 버튼 등의 정렬이 부자연스러움.
- **다음 작업 조치**: 고정 픽셀(px) 대신 상대 치수(vh, vw, %)를 도입하고, `BottomSheet.tsx` 내부 구조를 CSS Grid나 Flexbox gap 베이스로 리팩토링.

### #4 엣지(관계선) 클릭 팝업 이슈
- **증상**: 포커스가 없는 상태에서도 선을 누르면 팝업이 뜨도록 코드는 수정/배포되었으나, 실제로 캔버스에서 PointerEvent가 엣지에 정확하게 잡히지 않고 뒤의 노드가 잡히는 등의 클릭 판정 버그 의심.
- **다음 작업 조치**: `GraphCosmos.tsx`의 엣지 렌더러 히트박스(Hitbox) 두께 증가 설정 혹은 `onLinkClick` 로그 모니터링 후 Raycaster 우선순위 조정.

## ✅ 완료 및 배포 완료 내역 (해결됨)
1. **Vercel 빌드 무한 실패 해결**: `middleware.ts` 파일 자체를 완전히 제거하여 Edge Runtime 번들링 충돌(`middleware.js.nft.json` 누락 현상) 근본 원인 척결.
2. **우주맵 원본 사진 부활**: LOD(Level of Detail) 렌더 조건 완화 및 `onload` 재렌더 콜백 연결 완료.
3. **유저로그인 아이콘 가림 방어**: 우측 상단 공유 버튼과 아바타를 Flex Group으로 묶어 레이아웃 충돌 해결.
4. **iTunes 동명 곡/동명이인 오작동 방어**: 교차검증 알고리즘 추가 반영.
