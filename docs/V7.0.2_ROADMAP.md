## 1. 개요 (Overview)

V7.0.2의 핵심 목표는 두 가지입니다:

1. **대규모 아티스트 확장**: 33개 CSV 플레이리스트에서 **169명+ 한국 신규 아티스트**를 우주에 편입하고, MusicBrainz 기반으로 앨범/크레딧/피처링/컴필레이션 데이터를 확보
2. **"오늘의 우주" 기능**: 오늘 날짜에 역사적으로 발매된 앨범을 추천하고, 자율주행과 통합하여 "타임캡슐 워프" 경험을 제공

### 1.1 V7.0.1 → V7.0.2 달라지는 것

| 항목 | V7.0.1 | V7.0.2 |
|------|--------|--------|
| 아티스트 수 | 1,213명 | **~1,380명** (+169) |
| 앨범 발매일 데이터 | 없음 | **Supabase `album_releases` 테이블** |
| 오늘의 추천 | 없음 | **"오늘의 우주" (Today's Universe)** |
| 데이터 검증 주기 | 수동 | **매일 오전 2시 자동 검증 (CRON)** |
| 자율주행 시작점 | 유저 선택 필수 | **미선택 시 "오늘의 아티스트"에서 랜덤 출발** |
| Admin 앨범 관리 | 없음 | **캘린더형 발매일 뷰** |

## 2. 아티스트 확장 계획 (Artist Expansion)

### 2.1 데이터 수집 파이프라인

```
[33개 CSV 플레이리스트]
    ↓ scripts/v7.1-collect-new-artists.py
[신규 한국 아티스트 169명+ 추출]
    ↓
[MusicBrainz MBID 매핑 + 릴리즈(앨범/싱글/EP) 목록 수집]
    ↓ (rate limit: 1.1초/요청, 총 ~5분)
[public/data/new-artists-data.json 저장]
    ↓
[Ingest 스크립트로 우주에 편입]
    ├─ v5-layout.json에 새 노드 추가 (d3-force 좌표 계산)
    ├─ v5-edges.json에 엣지 추가 (피처링/프로듀서/작사가 등)
    └─ v5-details.json에 상세 정보 추가
```

### 2.2 데이터 수집 항목 (아티스트별)

| 항목 | 출처 | 활용 |
|------|------|------|
| MBID | MusicBrainz | 아티스트 고유 식별자 |
| 앨범 목록 | MusicBrainz release-groups | 디스코그래피 |
| **앨범 발매일** | MusicBrainz + 나무위키 + 위키피디아 | "오늘의 우주" |
| 곡별 크레딧 | MusicBrainz recording-credits | 엣지(WRITER, PRODUCER) 생성 |
| 피처링 정보 | MusicBrainz artist-rels | 엣지(FEATURED) 생성 |
| 컴필레이션 참여 | MusicBrainz release (compilation) | 엣지(INDIRECT) 생성 |
| 한글 이름 | CSV 원본 + 나무위키 | nameKo 필드 |
| 장르 | CSV Genres 필드 | accent 색상 결정 |

### 2.3 Spotify 발매일 신뢰성 경고

> ⚠️ **주의**: Spotify API의 `release_date`는 종종 **재발매/리마스터링 날짜**를 원본 발매일로 표시하여 데이터가 부정확합니다. 따라서:
> - **1차 출처**: MusicBrainz `first-release-date`
> - **2차 검증**: GPT-5 Nano를 이용한 나무위키/위키피디아 교차 확인
> - **3차 보정**: Admin 캘린더에서 수동 정정

## 3. 데이터베이스 확장 (New Tables)

```sql
-- 앨범 발매일 데이터 (아티스트별 앨범 디스코그래피)
CREATE TABLE album_releases (
  id            SERIAL PRIMARY KEY,
  artist_id     TEXT NOT NULL,          -- 우주 노드 ID (spotifyId 또는 override_xxx)
  artist_name   TEXT NOT NULL,          -- 아티스트 이름
  artist_name_ko TEXT,                  -- 한글 이름
  album_title   TEXT NOT NULL,
  album_type    TEXT DEFAULT 'Album',   -- Album, Single, EP, Compilation
  release_date  DATE NOT NULL,          -- 발매일 (YYYY-MM-DD)
  mbid          TEXT,                   -- MusicBrainz release-group ID
  source        TEXT DEFAULT 'musicbrainz', -- 출처: musicbrainz, namuwiki, wikipedia, manual
  verified      BOOLEAN DEFAULT false,  -- GPT-5 Nano 검증 완료 여부
  verified_at   TIMESTAMPTZ,
  verification_source TEXT,             -- 검증에 사용된 출처
  verification_note TEXT,               -- 검증 시 발견된 차이점 메모
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  updated_at    TIMESTAMPTZ DEFAULT NOW()
);

-- 날짜별 검증 현황 (CRON 작업 추적)
CREATE TABLE daily_verification_logs (
  id            SERIAL PRIMARY KEY,
  target_date   DATE NOT NULL,          -- 검증 대상 날짜 (오늘 또는 내일)
  total_albums  INT DEFAULT 0,          -- 해당 날짜의 총 앨범 수
  verified      INT DEFAULT 0,          -- 검증 완료 수
  corrected     INT DEFAULT 0,          -- 발매일 수정된 수
  status        TEXT DEFAULT 'pending', -- pending, running, completed, failed
  llm_model     TEXT DEFAULT 'gpt-5-nano',
  started_at    TIMESTAMPTZ,
  completed_at  TIMESTAMPTZ,
  log           TEXT,                   -- 실행 로그
  created_at    TIMESTAMPTZ DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_album_releases_date ON album_releases(release_date);
CREATE INDEX idx_album_releases_artist ON album_releases(artist_id);
CREATE INDEX idx_album_releases_month_day ON album_releases(EXTRACT(MONTH FROM release_date), EXTRACT(DAY FROM release_date));
```

## 4. "오늘의 우주" 기능 (Today's Universe)

### 4.1 핵심 개념

**"과거 오늘 날짜에 발매된 앨범들"**을 보여주는 타임캡슐 기능.

- 오늘이 3월 22일이면, **1950~2026년 3월 22일에 발매된** 모든 앨범을 표시
- 자율주행 미선택 시 → "오늘의 아티스트" 중 랜덤으로 하나 선택 → 자율주행 출발점

### 4.2 데이터 흐름

```
┌──────────────────────────────────────────────────────┐
│  매일 오전 2:00 (CRON → Vercel Cron Job)              │
│                                                      │
│  1. album_releases에서 내일 + 오늘 발매 앨범 조회     │
│  2. GPT-5 Nano로 각 앨범 발매일 교차 검증              │
│     └─ 프롬프트: "아티스트 X의 앨범 Y가 YYYY-MM-DD에   │
│        발매되었나? MusicBrainz, 나무위키, 위키피디아    │
│        정보를 바탕으로 확인해줘."                       │
│  3. 틀린 정보 발견 시 → album_releases UPDATE          │
│     └─ verification_note에 차이점 기록                 │
│  4. daily_verification_logs에 결과 기록                │
│                                                      │
└──────────────────────────────────────────────────────┘
             ↓
┌──────────────────────────────────────────────────────┐
│  유저 브라우저 (런타임)                                │
│                                                      │
│  GET /api/universe/today-albums?month=3&day=22       │
│     → album_releases에서 verified=true인 앨범만 반환  │
│     → { albums: [...], artistIds: [...] }            │
│                                                      │
│  자율주행 시작 시 (아티스트 미선택 상태):               │
│     → "오늘의 우주" 앨범 목록에서 랜덤 선택            │
│     → 해당 아티스트로부터 자율주행 출발                 │
│                                                      │
└──────────────────────────────────────────────────────┘
             ↓
┌──────────────────────────────────────────────────────┐
│  Admin 대시보드 (/admin/calendar)                     │
│                                                      │
│  📅 캘린더형 발매일 뷰                                │
│     → 날짜 클릭 → 해당일 발매 앨범 목록               │
│     → 검증 상태 (✅ 검증완료 / ⚠️ 미검증 / 🔄 수정됨) │
│     → 수동 발매일 정정 기능                           │
│                                                      │
│  📊 검증 현황 대시보드                                │
│     → daily_verification_logs 조회                    │
│     → 최근 7일 검증 결과 타임라인                     │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 4.3 GPT-5 Nano 검증 시스템 프롬프트

```
You are a K-Culture music data verifier.
Given an album release date claim, verify it using your knowledge of:
- MusicBrainz database
- Korean Wikipedia (나무위키)
- English Wikipedia

INPUT: { "artist": "IU", "album": "Palette", "claimed_date": "2017-04-21" }

OUTPUT (JSON only):
{
  "verified": true|false,
  "correct_date": "YYYY-MM-DD"|null,
  "confidence": 0.0-1.0,
  "source": "musicbrainz|namuwiki|wikipedia|multiple",
  "note_ko": "검증 결과 한국어 요약"
}

RULES:
- If the claimed_date is correct, set verified=true and correct_date=null
- If the date is wrong, set verified=false and provide the correct_date
- Only output JSON, no markdown
- confidence should reflect how certain you are (0.9+ for well-known albums)
```

### 4.4 비용 분석 (GPT-5 Nano)

| 항목 | 수치 |
|------|------|
| 하루 검증 대상 앨범 | ~20~50건 (365일 균등 분포 가정, 15,000 앨범 기준) |
| GPT-5 Nano 가격 | Input $0.10/1M tokens, Output $0.40/1M tokens |
| 검증 1건 평균 토큰 | ~200 input + ~100 output |
| 하루 비용 | 50건 × 300tok → ~$0.001 (**거의 무료**) |
| 월 비용 | **~$0.03** |

## 7. Vercel CRON Job 설계

### 7.1 API 엔드포인트

```
POST /api/cron/verify-albums
  ├─ x-vercel-cron-signature 헤더로 보안 검증
  ├─ 오늘 + 내일 날짜의 미검증 앨범 조회
  ├─ GPT-5 Nano로 각 앨범 교차 검증
  ├─ 결과 → album_releases UPDATE + daily_verification_logs INSERT
  └─ 최대 실행시간: 60초 (Vercel Hobby) / 300초 (Pro)
```

### 7.2 vercel.json CRON 설정

```json
{
  "crons": [
    {
      "path": "/api/cron/verify-albums",
      "schedule": "0 17 * * *"
    }
  ]
}
```

> ⚠️ Vercel CRON은 UTC 기준이므로, 한국시간 오전 2시 = UTC 17시 (전날)

### 7.3 Fallback 전략

| 상황 | 대응 |
|------|------|
| GPT-5 Nano 429 (Rate Limit) | 미검증 상태로 남김 → 다음날 CRON에서 재시도 |
| CRON 실행 실패 | daily_verification_logs에 status='failed' 기록, Admin에서 수동 실행 버튼 |
| 검증 결과 불확실 (confidence < 0.7) | verified=false 유지, verification_note에 "저신뢰: 수동 확인 필요" 기록 |

## 10. 기술 의사결정 기록

### Q: 왜 GPT-5 Nano를 검증에 사용하는가?

**A**: 
- 하루 50건 수준의 소량 검증 → **월 $0.03 미만** (거의 무료)
- Flash-Lite는 한국 음악 발매일 지식이 부족할 수 있음
- GPT-5 Nano는 가장 저렴하면서도 사실 확인(Fact-checking)에 충분한 성능
- MusicBrainz와 나무위키/위키피디아 데이터를 교차 확인하므로, LLM이 "정보 종합자" 역할만 하면 됨

### Q: 왜 Spotify 발매일을 안 쓰는가?

**A**:
- Spotify는 재발매/리마스터 시 원본 발매일을 덮어써버리는 경우가 많음
- 예: 이문세의 1985년 앨범이 2020년 리마스터로 `release_date: 2020-01-15`로 표시
- MusicBrainz의 `first-release-date`가 원본 발매일에 가장 정확

### Q: CRON이 실패하면?

**A**:
- `daily_verification_logs`에 `status='failed'` 기록
- Admin 대시보드에서 경고 배너 표시
- 수동 실행 버튼으로 즉시 재시도 가능
- 미검증 앨범은 "오늘의 우주"에 ⚠️ 표시로 노출 (숨기지 않음)
